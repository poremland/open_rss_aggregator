# Conductor Workflow

## Task Workflow
Every task in a track must follow this lifecycle:

1. **Implementation**
   - Implement the changes according to the task description.
   - Adhere to the project's tech stack and coding standards.
2. **Testing**
   - Add or update tests (RSpec) for the new functionality.
   - Ensure all tests pass.
3. **Validation**
   - Manually verify the changes if applicable.
   - Run existing test suites to ensure no regressions.
4. **Commit**
   - Commit the changes with a clear, descriptive message.

## Standards
- Follow Ruby on Rails best practices.
- Ensure proper error handling and logging.
- Maintain consistency with existing codebase patterns.
